# 0.1.32 代码优化设计

## 目标

本轮优化只收敛两个已经影响用户体验的问题：

- 大模型思考模式开启后，连通性测试可能只返回 reasoning 内容，导致误判为空响应。
- 首页成功提示和纵向卡片堆叠容易撑出滚动条，桌面工具观感不稳定。

## 方案

### LLM 连通性测试

保留现有 OpenAI 兼容请求结构，不新增依赖，不引入服务商分支。

- 将连接测试的 `max_tokens` 从极小值提高到固定常量，给 reasoning 响应留出空间。
- 判断测试响应时同时接受 `content` 和 `reasoning_content`，避免思考模型返回 reasoning-only 时误报失败。
- UI 上把“思考模式”提升到大模型配置区，并补充三语言提示，说明语音润色通常不需要开启。

### 首页布局

保持当前视觉风格，不重构应用外壳。

- 首页不再展示完成后的识别正文回执，避免保留正文并避免额外占高。
- 首页隐藏外层标题行，让语音输入卡成为首屏主入口。
- 将“启动方式”和“最近使用”在宽窗口下并排展示，窄窗口自动回到单列。
- 对状态说明和功能标签做行数/溢出保护，降低文案变化导致滚动条出现的概率。

## 风险

- 真正阻断主流程的错误卡、ASR 未配置提示仍会占用高度；这是必要反馈，不应隐藏。
- 普通浏览器无法完全替代 Tauri WebView 验证，发布前仍需手工打开桌面窗口检查首页。

## 验收

- `npm run ai:release-check` 通过。
- `npx tauri build` 能生成 Windows NSIS 安装包。
- 首页空闲、录音中、成功后、三个触发方式开启时不出现非必要滚动条。
