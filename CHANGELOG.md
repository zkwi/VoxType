# 更新日志

这里记录 VoxType 的重要变更。

格式参考 [Keep a Changelog](https://keepachangelog.com/en/1.1.0/)。

## [未发布]

## [0.1.36] - 2026-04-27

### 变更

- 发现新版本的提示增加“立即更新”按钮，可直接启动下载并自动安装流程。
- 非编辑区域屏蔽 WebView 默认右键菜单，避免首页弹出 Refresh、More Tools 等浏览器菜单。

## [0.1.35] - 2026-04-27

### 变更

- 首页移除语音输入卡片标题，并统一“启动方式”和“输入表现”的标题样式，减少图标和标题层级噪音。
- 完成输入卡新增“复制文本”按钮，方便用户直接复制最近一次识别文本。

### 技术实现

- 新增专用的最近输入文本复制命令，复用现有剪贴板写入校验逻辑，不触发粘贴、不记录识别正文。
- 收紧首页复制状态处理，避免复制过程中开始新录音时误更新旧结果状态。

## [0.1.34] - 2026-04-27

### 变更

- 首页改为“语音输入全宽、最近输入与启动方式并排、输入表现全宽”的结构，恢复最近一次输入内容入口，同时避免压缩统计表现区域。
- 首页普通状态下隐藏内容区滚动条，仅在配置缺失或错误详情需要展示时允许滚动，减少日常使用中的额外滚动干扰。

### 修复

- 修复 Windows 启动时同步开机自启动注册表可能闪现黑色控制台窗口的问题。

### 审计

- 复核最近一次输入内容仅保留在前端窗口内存，不写入统计文件或日志。
- 补齐 v0.1.33 更新日志记录，避免发布历史断档。

## [0.1.33] - 2026-04-27

### 变更

- 重构热词、API 配置和选项页的信息层级：默认展示必要和高频设置，高级区域承载低频排障项，底层实现参数保留 `config.toml` 支持。
- 大模型 API 配置默认保留核心字段和测试连接，将超时、thinking 等低频参数移入高级设置。
- 热词页保留常用热词和润色提示词入口，将最近上下文和自动热词候选收进高级设置。
- 选项页默认聚焦快捷键、粘贴方式、麦克风、字幕外观、开机启动和关闭行为。

### 修复

- 修复自动更新静默安装后不会自动打开新版本的问题，NSIS 安装器改用 `/S /R` 参数。
- 修复隐藏的 config-only 高级字段校验错误阻断可见设置保存的问题，未修改的隐藏字段会保留原值。

### 测试

- 补充更新安装参数测试，确认静默安装后会请求重新打开应用。
- 补充隐藏高级配置字段保存兼容性测试。

## [0.1.32] - 2026-04-27

### 变更

- 大模型 API 配置页将“思考模式”开关放到大模型配置区，并补充三语言说明，提示语音润色通常不需要开启。
- 首页移除完成输入后的正文回执卡，减少首页占高，并避免在主窗口继续保留识别正文。
- 首页隐藏外层标题行，将启动方式和最近使用改为宽窗口双栏展示，减少普通状态下的额外滚动条。
- 首页状态说明和功能标签增加溢出保护，降低文案变长导致布局被撑高的概率。

### 修复

- 修复部分思考模型在连通性测试中只返回 reasoning 内容时被误判为空响应的问题。

### 测试

- 补充 LLM 连通性测试响应解析单元测试，覆盖 reasoning-only 响应、空响应和测试 token 上限。

## [0.1.31] - 2026-04-26

### 变更

- 应用内更新下载 NSIS 安装包后改为静默启动安装，并自动退出当前版本释放文件，减少用户在更新流程中反复点击“下一步”或卸载确认。
- 保存设置和启动阶段将开机自启动同步、全局热键重启等慢副作用移到后台执行，减少切换设置项或启动时的界面卡顿。
- 选项页默认展示软件更新区块和启动时自动检查更新开关，仅将 GitHub Release Repo 等排障参数保留在高级设置中。
- 自动粘贴前再次确认剪贴板仍是本次识别文本，若被改写会重新写入；同时提高恢复原剪贴板前的安全等待，减少慢应用粘贴到旧剪贴板内容的问题。
- 将剪贴板恢复延迟默认值调整为 1800ms，并在自动粘贴时对更低配置使用安全下限。
- 新增“自动移除句末句号”选项，默认保持开启；关闭后保留 ASR 或大模型输出的句末标点。

### 测试

- 补充配置保存副作用判断测试，确认无关设置不会触发热键重启或开机启动同步。
- 补充更新安装参数测试，确认应用内更新使用 NSIS 静默安装参数。
- 补充剪贴板输出时序测试，覆盖粘贴前文本匹配判断和恢复延迟安全下限。
- 补充最终文本句末标点开关测试，覆盖开启移除和关闭保留两种行为。

## [0.1.30] - 2026-04-26

### 变更

- 将前端 session 阶段判断、quiet warning 判断和前端诊断日志工具抽为独立纯函数模块，减少 `VoxTypeController` 的策略代码。
- 移除最近输入结果中未展示的错误分支状态，错误继续由首页错误卡片统一承载。
- 将配置校验逻辑从 `config.rs` 拆入 `config_validation.rs`，让配置模型、IO 和字段校验职责更清晰。
- 新增轻量错误上下文工具，先收敛配置与最近上下文路径中的重复错误字符串拼接。

### 测试

- 补充 `recording` 阶段 `toggle` 仍可进入停止流程的回归测试。

## [0.1.29] - 2026-04-26

### 新增

- 首页新增最近一次输入完成提示，成功后保留 `Ctrl + V` 兜底说明。
- 首页错误提示新增可执行操作按钮，支持重试、跳转 API 配置、打开选项、打开日志和复制诊断报告。

### 修复

- 录音 `starting` / `stopping` 阶段加入重复触发保护，减少连续点击或连续热键造成的状态困惑。
- 主界面未知 Tauri 调用错误改为用户友好提示，原始错误仅保留在日志中。
- 移除页面内容区重复展示的 `VoxType` 标题，修复首页、选项和统计页的标题排版噪音。
- 修复 Windows lib 单元测试缺少 common controls v6 manifest 时无法启动的问题。

### 测试

- 补充 `starting` / `stopping` 阶段 `toggle` 防重复触发单元测试。

## [0.1.28] - 2026-04-26

### 修复

- 收紧诊断报告本地路径脱敏边界，避免相似用户目录前缀被误判为用户目录。

### 变更

- 将诊断路径脱敏逻辑抽为可单独测试的纯函数，降低隐私相关代码的维护成本。

### 测试

- 补充路径脱敏单元测试，覆盖用户目录精确匹配、子路径匹配和相似前缀不应脱敏的场景。

## [0.1.27] - 2026-04-26

### 修复

- 修复手动检查更新失败时不再显示错误提示的回归，保持拆分前的用户反馈行为。

### 变更

- 整理主应用控制器 import 结构，减少上轮拆分后留下的阅读噪音。
- 复核主应用重构后的文件规模，`VoxTypeController` 继续保持在 50 KiB 以下。

## [0.1.26] - 2026-04-26

### 变更

- 将主应用状态与命令处理迁移到 `VoxTypeController`，让 `VoxTypeApp` 保持为轻量渲染组件。
- 拆出悬浮字幕、自动热词、更新、诊断和窗口控制器，继续降低主应用组件复杂度。
- 将麦克风音量条计算抽为前端工具函数，减少界面控制器中的展示细节。

### 测试

- 补充健康检查单元测试，确认隐私、额外触发方式和系统静音等软选项不会阻断 ready 状态。
- 补充触发方式摘要单元测试，覆盖多触发方式展示和未启用状态。

## [0.1.25] - 2026-04-26

### 变更

- 将 `+page.svelte` 降为轻量路由入口，主应用拆为 `VoxTypeApp`、`AppShell`、`AppContent` 和全局样式组件。
- 拆出首页、热词、API 配置、选项和统计页面级组件，降低路由入口与主模板复杂度。
- 将设置表单、首页卡片和悬浮/弹窗相关样式迁移到对应组件内，减少路由文件样式负担。

### 测试

- 补充健康检查状态单元测试，覆盖缺少 ASR 密钥、无麦克风、无触发方式和 ready 状态。

## [0.1.24] - 2026-04-26

### 新增

- 新增页面级高级设置开关，热词、API 配置和选项页可分别展开高级内容。
- 新增发布前本地检查入口 `npm run ai:release-check`，统一执行本地检查、clippy 和 Tauri debug 构建。

### 变更

- 热词页默认只展示手动热词，场景上下文、大模型 Prompt 和自动热词候选移入高级设置。
- API 配置页突出豆包 ASR 必填认证，并将大模型 API 标记为可选配置。
- 选项页默认只展示日常设置，剪贴板、录音、更新和诊断参数移入高级设置。
- 首页健康检查支持 warning 等级，非阻断注意事项改为折叠展示。
- CHANGELOG 改为简体中文维护。

### 修复

- 修正 API 配置页与选项页共用高级设置开关导致跨页互相影响的问题。

## [0.1.23] - 2026-04-26

### 新增

- 新增独立的“已接受自动热词”列表，让生成热词与用户维护的自定义热词在界面上保持区分。

### 变更

- ASR 请求、LLM 润色提示词、提示词预览和自动热词去重过滤改为使用合并后的有效热词列表，但配置中仍保留分开的存储列表。
- 优化悬浮字幕布局，音量指示不再遮挡多行字幕文本，浏览器换行也不会额外制造视觉错行。

## [0.1.22] - 2026-04-26

### 新增

- 新增 Windows GitHub Actions CI 流程，覆盖前端检查、密钥扫描、Rust 格式检查、clippy 和测试。

### 变更

- 将共享前端应用类型从 `src/routes/+page.svelte` 移到 `src/lib/types/app.ts`，作为页面和组件拆分的第一步低风险整理。
- 为 ASR 空结果处理、处理中阶段会话切换、非法音频采样率校验补充聚焦的 Rust 回归测试。
- 为高风险的配置、会话和剪贴板输出边界补充文档注释。
- 新增结构化 ASR warning code，让低优先级剪贴板恢复提示可以不依赖本地化文本匹配来隐藏。

### 修复

- 默认隐藏较大的“部分剪贴板格式未备份”提示，不再显示到悬浮字幕和主窗口 toast，但仍保留日志中的诊断 warning。

## [0.1.21] - 2026-04-26

### 变更

- 自动热词状态不再向前端暴露本地历史文件路径。
- 自动热词生成失败时，不再把服务原始响应正文透传到错误处理路径。
- 热词历史测试现在会清理临时目录。
